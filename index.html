Groovy to Java8

Why?

  • Better quality
  • Faster to develop

Groovy (good things)

  • Less code
  • Easier to read
  • Grails

Grails (good things)

  • Same scheme for all projects
  • Most important things out of box

Groovy (bad things)

  • Code style is very important
  • Harder to maintain / refactor
  • Bad IDE support
  • Slower to compile / start
  • Probably dead (last release 2+ years ago)

Grails (bad things)

  • Good idea / bad implementation
  • Slow starts -> slow developing (10s rule)
  • Hard to trust while developing
  • Probably dead

Java (bad things)

  • A lot of code / hard to read
  • Slow starts (<10s)
  • Overcomplexity (approaches)

Java (good things)

  • Good IDE support
  • Static compilation -> easier to refactor
  • Faster compiles / starts -> fix / restart cycle

Java8 (good things)

  • Lambda
  • Streams
  • Method references
  • Default methods

Groovy to Java 8

  • Closures -> lambda
  • Boilerplate -> Lombok
  • Collections / Maps -> streams + soyuz-is-to
  • Convert / Files -> soyuz-is-to + Guava + Apache commons
